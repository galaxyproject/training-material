Introduction

In this tutorial, we perform proteogenomic database searching using the Mass Spectrometry data. The inputs for performing the proteogenomic database searching are the peaklist MGF files and the FASTA database file. The FASTA database is obtained by running the first workflow ‚ÄúUniprot_cRAP_SAV_indel_translatedbed.FASTA‚Äù. The second workflow focuses on performing database search of the peak list files (MGFs).

Match peptide sequences

The search database labelled Uniprot_cRAP_SAV_indel_translatedbed.FASTA is the input database that will be used to match MS/MS to peptide sequences via a sequence database search.

For this, the sequence database-searching program called SearchGUI will be used.The generated dataset collection of the three MGF files in the history is used as the MS/MS input. We will walk through a number of these settings in order to utilize SearchGUI on these example MGF files.

comment Comment:

Note that sequence databases used for proteogenomics are usually much larger than the excerpt used in this tutorial. When using large databases, the peptide identification step can take much more time for computation. In proteogenomics, choosing the optimal database is a crucial step of your workflow.

PeptideShaker

PeptideShaker is a post-processing software tool that processes data from the SearchGUI software tool. It serves to organize the Peptide-Spectral Matches (PSMs) generated from SearchGUI processing and is contained in the SearchGUI archive. It provides an assessment of confidence of the data, inferring proteins identified from the matched peptide sequences and generates outputs that can be visualized by users to interpret results. PeptideShaker has been wrapped in Galaxy to work in combination with SearchGUI outputs.

comment Comment: File Formats

There are a number of choices for different data files that can be generated using PeptideShaker. A compressed file can be made containing all information needed to view them results in the standalone PeptideShaker viewer. A mzidentML file can be generated that contains all peptide sequence matching information and can be utilized by compatible downstream software. Other outputs are focused on the inferred proteins identified from the PSMs, as well as phosphorylation reports, relevant if a phosphoproteomics experiment has been undertaken.

    • param-check ‚ÄúInclude the protein sequences in mzIdentML‚Äù: No
    • param-check ‚ÄúOutput options‚Äù: Select the PSM Report (Peptide-Spectral Match) and the Certificate of Analysis

Create database

The mzidentml output from the Peptide shaker is converted into an sqlite database file by using the mz to sqlite tool. This sqlite output is used to open the Multi-omics visualization platform, wherein you can view the spectra of the peptides using Lorikeet parameters. To open the MVP viewer, click on the ‚ÄúVisualize in MVP Application‚Äù icon ( this will pop-open the interactive multi-omics viewer in a new window/tab)

Merged Uniprot and cRAP database

The file named ‚ÄúTrimmed_ref_500_Uniprot_cRAP.fasta‚Äù is the trimmed version of Uniprot and cRAP database merged Fasta files.

This Fasta file will be subjected to few text manipulation steps in order to get the tabular file for the known peptides. The known peptides are those which are already annotated in the reference database, in this case the Uniprot database. The first step is to convert this FASTA file to tabular in order to proceed with text manipulation.

ORDER BY psms.id

BLASTP (Basic Local Alignment Search Tool- proteins)

BLAST is a web based tool used to compare biological sequences. BlastP, matches protein sequences against a protein database. More specifically, it looks at the amino acid sequence of proteins and can detect and evaluate the amount of differences between say, an experimentally derived sequence and all known amino acid sequences from a database. It can then find the most similar sequences and allow for identification of known proteins or for identification of potential peptides associated with novel proteoforms.

BlastP search is carried out with the PSM report (output from PeptideShaker). Before, BlastP analysis the ‚ÄúPeptides_for_Blast-P_analysis‚Äù is first converted from Tabular format to FASTA file format which can be easily read by the BlastP algorithm. This is done with the help of ‚ÄúTabular to FASTA‚Äù conversion tool. The short BlastP uses parameters for short peptide sequences (8-30 aas). Please use the rerun option to look at the parameters used.

hands_on Hands-on: Tabular to FASTA

  1. Tabular to FASTA tool: with the following parameters:
    • ‚ÄúTitle column‚Äù: 1
    • ‚ÄúSequence Column‚Äù:2

The output FASTA file is going to be subjected to BLAST-P analysis.

hands_on Hands-on: NCBI BLAST+ blastp

  1. NCBI BLAST+ blastp tool: Run BLASTP with:
    • param-file ‚ÄúProtein query sequence(s)‚Äù: Data input 'query' (fasta)
    • param-select ‚ÄúSubject database/sequences‚Äù: Locally installed BLAST database
      • param-select ‚ÄúProtein Blast database‚Äù: Select nr_mouse_current
    • param-check ‚ÄúType of BLAST‚Äù: blastp-short - BLASTP optimized for queries shorter than 30 residues
    • param-text ‚ÄúSet expectation value cutoff‚Äù:200000.0
    • param-select ‚ÄúOutput format‚Äù: Tabular(extended 25 columns)
    • ‚ÄúAdvanced Options‚Äù: Show Advanced Options
      • param-check ‚ÄúFilter out low complexity regions (with SEG)‚Äù: No
      • param-select ‚ÄúScoring matrix and gap costs‚Äù: PAM30
        • param-select ‚ÄúGap Costs‚Äù: Existence: 9 Extension: 1
      • param-text ‚ÄúMaximum hits to show‚Äù: 1
      • param-text ‚ÄúMaximum number of HSPs (alignments) to keep for any single query-subject pair‚Äù:1
      • param-text ‚ÄúWord size for wordfinder algorithm‚Äù: 2
      • param-text ‚ÄúMultiple hits window size‚Äù: 40
      • param-text ‚ÄúMinimum score to add a word to the BLAST lookup table‚Äù: 11
      • param-select ‚ÄúComposition-based statistics‚Äù: 0 (No composition)
      • param-check ‚ÄúShould the query and subject defline(s) be parsed?‚Äù: No
      • param-select ‚ÄúRestrict search of database to a given set of ID‚Äôs‚Äù:No restriction

      comment Comment:

      This feature provides a means to exclude ID‚Äôs from a BLAST database search. The expectation values in the BLAST results are based upon the sequences actually searched, and not on the underlying database. Note this cannot be used when comparing against a FASTA file.

    • param-text ‚ÄúMinimum query coverage per hsp (percentage, 0 to 100)‚Äù: 0
    • param-check ‚ÄúCompute locally optimal Smith-Waterman alignments‚Äù:No

Once BlastP search is performed, it provides with a tabular output containing peptides corresponding to novel proteoforms termed as ‚ÄúNovel peptides‚Äù. Now this output is further processed by comparing the novel peptide output with the PSM report for selecting only distinct peptides which pass these criteria. This could be achieved by proceeding to the novel peptide analysis tutorial.
