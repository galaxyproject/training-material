Introduction

In this tutorial, we perform proteogenomic database searching using the Mass Spectrometry data. The inputs for performing the proteogenomic database searching are the peaklist MGF files and the FASTA database file. The FASTA database is obtained by running the first workflow ‚ÄúUniprot_cRAP_SAV_indel_translatedbed.FASTA‚Äù. The second workflow focuses on performing database search of the peak list files (MGFs).

Match peptide sequences

The search database labelled Uniprot_cRAP_SAV_indel_translatedbed.FASTA is the input database that will be used to match MS/MS to peptide sequences via a sequence database search.

For this, the sequence database-searching program called SearchGUI will be used.The generated dataset collection of the three MGF files in the history is used as the MS/MS input. We will walk through a number of these settings in order to utilize SearchGUI on these example MGF files.

Comment: Tool Versions

The tools are subjected to changes while being upgraded. Here we have used an older version of SearchGUI (v3.3.10.1)-Peptide Shaker (v1.16.36.3). The new versions are available in Galaxy.

SearchGUI

Hands-on: SearchGUI
  1. Search GUI Tool: toolshed.g2.bx.psu.edu/repos/galaxyp/peptideshaker/search_gui/3.3.10.1 with the following parameters:

    • param-file ‚ÄúProtein Database‚Äù: Uniprot_cRAP_SAV_indel_translatedbed.FASTA (Or however you named the FASTA file)

      Comment

      The ‚ÄúUniprot_cRAP_SAV_indel_translatedbed‚Äù FASTA database is obtained when you run the first proteogenomics workflow. Please make sure to run the 1st workflow.

    • param-files ‚ÄúInput Peak lists (mgf)‚Äù: MGF files dataset collection.

      • Click the Dataset collection icon on the left of the input field:

      • Select the appropriate dataset collection from the list

    • Section Search Engine Options:

      • param-check ‚ÄúDB-Search Engines‚Äù: X!Tandem

        Comment

        The section Search Engine Options contains a selection of sequence database searching programs that are available in SearchGUI. Any combination of these programs can be used for generating PSMs from MS/MS data. For the purpose of this tutorial, X!Tandem we will be used.

    • Section Protein Digestion Options:
      • param-select ‚ÄúEnzyme‚Äù: Trypsin
      • param-text ‚ÄúMaximum Missed Cleavages‚Äù: 2
    • Section Precursor options:
      • param-select ‚ÄúPrecursor Ion Tolerance Units‚Äù: Parts per million (ppm)
      • param-text ‚ÄúPrecursor Ion Tolerance‚Äù: 10
      • param-text ‚ÄúFragment Tolerance (Daltons)‚Äù: 0.05 (this is high resolution MS/MS data)
      • param-text ‚ÄúMinimum charge‚Äù:2
      • param-text ‚ÄúMaximum charge‚Äù:6
      • param-select ‚ÄúForward Ion‚Äù: b
      • param-select ‚ÄúReverse Ion‚Äù: y
      • param-text ‚ÄúMinimum Precursor Isotope‚Äù :0
      • param-text ‚ÄúMaximum Precursor Isotope‚Äù :1
    • Section Protein Modification Options:
      • param-select ‚ÄúFixed Modifications‚Äù: Carbamidomethylation of C, ITRAQ-4Plex of K, ITRAQ-4Plex of Ntermini

      • param-select ‚ÄúVariable modifications‚Äù: Oxidation of M, ITRAQ-4Plex of Y

        For selection lists, typing the first few letters in the window will filter the available options.

    • Section Advanced Options:
      • param-select ‚ÄúX!Tandem Options‚Äù: Advanced
        • param-check ‚ÄúX!Tandem: Quick Acetyl‚Äù: No
        • param-check ‚ÄúX!Tandem: Quick Pyrolidone‚Äù: No
        • param-check ‚ÄúX!Tandem: Protein stP Bias‚Äù: No
        • param-text ‚ÄúX!Tandem: Maximum Valid Expectation Value‚Äù: 100

Once the database search is completed, the SearchGUI tool will output a file (called a SearchGUI archive file) that will serve as an input for the next section, PeptideShaker. Rename the output as ‚ÄúCompressed SearchGUI results‚Äù

Comment

Note that sequence databases used for proteogenomics are usually much larger than the excerpt used in this tutorial. When using large databases, the peptide identification step can take much more time for computation. In proteogenomics, choosing the optimal database is a crucial step of your workflow.

PeptideShaker

PeptideShaker is a post-processing software tool that processes data from the SearchGUI software tool. It serves to organize the Peptide-Spectral Matches (PSMs) generated from SearchGUI processing and is contained in the SearchGUI archive. It provides an assessment of confidence of the data, inferring proteins identified from the matched peptide sequences and generates outputs that can be visualized by users to interpret results. PeptideShaker has been wrapped in Galaxy to work in combination with SearchGUI outputs.

Comment: File Formats

There are a number of choices for different data files that can be generated using PeptideShaker. A compressed file can be made containing all information needed to view them results in the standalone PeptideShaker viewer. A mzidentML file can be generated that contains all peptide sequence matching information and can be utilized by compatible downstream software. Other outputs are focused on the inferred proteins identified from the PSMs, as well as phosphorylation reports, relevant if a phosphoproteomics experiment has been undertaken.

Hands-on: PeptideShaker
  1. Peptide Shaker Tool: toolshed.g2.bx.psu.edu/repos/galaxyp/peptideshaker/peptide_shaker/1.16.36.3 with the following parameters:
    • param-file ‚ÄúCompressed SearchGUI results‚Äù: The SearchGUI archive file
    • param-select ‚ÄúSpecify Advanced PeptideShaker Processing Options‚Äù: Default Processing Options
    • param-select ‚ÄúSpecify Advanced Filtering Options‚Äù: Default Filtering Options
    • param-check ‚ÄúInclude the protein sequences in mzIdentML‚Äù: No
    • param-check ‚ÄúOutput options‚Äù: Select the PSM Report (Peptide-Spectral Match) and the Certificate of Analysis
    Comment: Certificate of Analysis

    The ‚ÄúCertificate of Analysis‚Äù provides details on all the parameters used by both SearchGUI and PeptideShaker in the analysis. This can be downloaded from the Galaxy instance to your local computer in a text file if desired.

  2. Inspect galaxy-eye the resulting files

A number of new items will appear in your History, each corresponding to the outputs selected in the PeptideShaker parameters. The Peptide Shaker‚Äôs PSM report is used as an input for the BlastP analysis. Before performing BlastP analysis. The Query Tabular tool and few test manipulation tools are used to remove spectra that belongs to the reference proteins. The output tabular file ‚ÄúPeptides_for_Blast-P_analysis‚Äù will contain only those spectra that did not belong to any known proteins.

Create a SQLite database for peptide, protein and genomic annotation visualization

The mzidentml output from the Peptide shaker is converted into an sqlite database file by using the mz to sqlite tool. This sqlite output is used to open the Multi-omics visualization platform, wherein you can view the spectra of the peptides using Lorikeet parameters. To open the MVP viewer, click on the ‚ÄúVisualize in MVP Application‚Äù icon ( this will pop-open the interactive multi-omics viewer in a new window/tab)

Hands-on: mz to sqlite

This tool extracts mzidentml and its associated proteomics datasets into a sqlite db

  1. mz to sqlite Tool: toolshed.g2.bx.psu.edu/repos/galaxyp/mz_to_sqlite/mz_to_sqlite/2.0.4+galaxy1 with the following parameters:
    • param-file ‚ÄúProteomics identification files‚Äù: PeptideShaker_mzidentml
    • param-file ‚ÄúProteomics Spectrum files‚Äù: Mo_Tai_MGFs
    • param-file ‚ÄúProteomics Search Database Fasta‚Äù: Uniprot_cRAP_SAV_indel_translatedbed.FASTA

The next step is to remove known peptides from the list of PSM‚Äôs that we acquired from the Peptide Shaker results. For that we need to perform Query tabular to extract list of known peptides from the UniProt and cRAP database.

Query Tabular

Hands-on: Remove Reference proteins
  1. Query Tabular tool with the following parameters:
  • param-repeat Insert Database Table (b): PSM report
    • Section Filter Dataset Input:
      • param-repeat Insert Filter Tabular Input Lines
        • ‚ÄúFilter by‚Äù: skip leading lines
        • ‚ÄúSkip lines‚Äù: 1
    • Section Table Options:
      • ‚ÄúSpecify Name for Table‚Äù: psms
      • ‚ÄúUse first line as column names‚Äù : No
      • ‚ÄúSpecify Column Names (comma-separated list)‚Äù:id,Proteins,Sequence
      • ‚ÄúOnly load the columns you have named into database‚Äù: Yes
      • param-repeat Table Index
        • ‚ÄúTable Index‚Äù: No
        • ‚ÄúIndex on Columns‚Äù: id

  • param-repeat Insert Database Table (c):PSM report

    • Section Filter Dataset Input
      • param-repeat Insert Filter Tabular Input Lines
        • ‚ÄúFilter by‚Äù: skip leading lines
        • ‚ÄúSkip lines‚Äù: 1
      • param-repeat Insert Filter Tabular Input Lines
        • ‚ÄúFilter by‚Äù: select columns
        • ‚ÄúEnter column numbers to keep‚Äù: 1,2
      • param-repeat Insert Filter Tabular Input Lines
        • ‚ÄúFilter by‚Äù: normalize list columns,replicate rows for each item in the list
        • ‚ÄúEnter column numbers to normalize‚Äù: 2
        • ‚ÄúList item delimiter in column‚Äù: ,
    • Section Table Options:
      • ‚ÄúSpecify Name for Table‚Äù: prots
      • ‚ÄúUse first line as column names‚Äù : No
      • ‚ÄúSpecify Column Names (comma-separated list)‚Äù:id,prot
      • ‚ÄúOnly load the columns you have named into database‚Äù: Yes
      • param-repeat Insert Table Index:
        • ‚ÄúTable Index‚Äù: No
        • ‚ÄúIndex on Columns‚Äù: prot,id
  • param-repeat Insert Database Table (a): Reference_Protein_Acessions
    • Section Table Options:
      • ‚ÄúTabular Dataset for Table‚Äù: Uniprot
      • ‚ÄúUse first line as column names‚Äù : No
      • ‚ÄúSpecify Column Names (comma-separated list)‚Äù:prot
      • param-repeat Insert Table Index:
        • ‚ÄúTable Index‚Äù: No
        • ‚ÄúIndex on Columns‚Äù: prot
      Comment

      By default, table columns will be named: c1,c2,c3,‚Ä¶,cn (column names for a table must be unique). You can override the default names by entering a comma separated list of names, e.g. ,name1,,,name2 would rename the second and fifth columns. Check your input file to find the settings which best fits your needs.

    • ‚ÄúSave the sqlite database in your history‚Äù: No

      Comment: Querying an SQLite Database

      Query Tabular can also use an existing SQLite database. Activating Save the sqlite database in your history will store the generated database in the history, allowing to reuse it directly.

    • ‚ÄúSQL Query to generate tabular output‚Äù: 
      SELECT psms.*
FROM psms
WHERE psms.id NOT IN
(SELECT distinct prots.id
FROM prots JOIN uniprot ON prots.prot = uniprot.prot)
ORDER BY psms.id
    • ‚Äúinclude query result column headers‚Äù: Yes
  • Click Run Tool and inspect the query results file after it turned green.

The output from this step is that the resultant peptides would be those which doesn‚Äôt belong in the Uniprot or cRAP database. The query tabular tool is used again to create a tabular output containing peptides ready for Blast P analysis.

Hands-on: Query Tabular

Query Tabular tool: with the following parameters:

  • Section Filter Dataset Input
    • param-repeat ‚ÄúInsert Filter Tabular Input Lines‚Äù
      • ‚ÄúFilter by‚Äù: skip leading lines
      • ‚ÄúSkip lines‚Äù: 1
  • Section Table Options:
    • ‚ÄúSpecify Name for Table‚Äù: psm
    • ‚ÄúUse first line as column names‚Äù : No
    • ‚ÄúSpecify Column Names (comma-separated list)‚Äù:id,Proteins,Sequence
    • ‚ÄúOnly load the columns you have named into database‚Äù: Yes
  • ‚ÄúSQL Query to generate tabular output‚Äù: 
    SELECT Sequence || ' PSM=' || group_concat(id,',') || ' length='
|| length(Sequence) as "ID",Sequence
FROM  psm
WHERE length(Sequence) >6
AND length(Sequence) <= 30
GROUP BY Sequence
ORDER BY length(Sequence),Sequence

  • ‚Äúinclude query result column headers‚Äù: Yes

  • Click Run Tool and inspect the query results file after it turned green.

Getting data Blast-P ready

BlastP search is carried out with the PSM report (output from PeptideShaker). Before, BlastP analysis the ‚ÄúPeptides_for_Blast-P_analysis‚Äù is first converted from Tabular format to FASTA file format which can be easily read by the BlastP algorithm. This is done with the help of ‚ÄúTabular to FASTA‚Äù conversion tool. The short BlastP uses parameters for short peptide sequences (8-30 aas). Please use the rerun option to look at the parameters used.

Hands-on: Tabular to FASTA (version 1.1.1)
  1. Tabular-to-FASTA tool: with the following parameters:
    • ‚ÄúTitle column‚Äù: 1
    • ‚ÄúSequence Column‚Äù:2

The output FASTA file is going to be subjected to BLAST-P analysis.

Comment: Tool Versions

The tools are subjected to changes while being upgraded. Thus, running the workflow provided with the tutorial, the user might need to make sure they are using the latest version including updating the parameters.

Conclusion

This completes the walkthrough of the proteogenomics database search workflow. This tutorial is a guide to perform database searching with mass spectronetry files and have peptides ready for Blast-P analysis, you can perform follow up analysis using the next GTN ‚ÄúProteogenomics Novel Peptide Analysis‚Äù. Researchers can use this workflow with their data also, please note that the tool parameters, reference genomes and the workflow will be needed to be modified accordingly.
