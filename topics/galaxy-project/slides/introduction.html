- The context in which Galaxy originated, in two thousand and five, helps explain a lot. - Sequencing was expensive, but provided valuable scientific insights. - Technical skills needed to analyse raw data correctly, were in very short supply - Investigators were impatient for results to publish from expensive data. - If an analysis cannot be scrutinised, the results cannot gain scientific trust. - Galaxy was our response to this growing shortfall in reliable analysis capacity. - Not only in genome sciences, but a risk to scientific integrity in many rapidly growing, data-rich fields. --- ## Solution: Accessible analysis using shareable and repeatable computation. - Galaxy was designed as a solution to the growing need for reliable, shareable analysis. - Integrate sharing and computational reproducibility into a generic analysis framework. - XML wrapper turns any Linux command line package into an interoperable tool - Wrapped packages gain framework benefits, such as workflow inter operability - ToolShed repository automates tool wrapper installation for server administrators. - Many "Flavours" are available for a framework instance. - Adaptable to any kind of open science data where packages are available to wrap. ??? - Galaxy was created to make open science analyses, reproducible and transparent. - Solve those tough challenges once, in a generic scientific workflow application. - Build those core scientific requirements into every analysis and output. - Accommodate any command line analysis package as an inter operable tool, to suit any scientist. - Make it easy to wrap packages, and install as tools, into that generic framework. - Domain specific tools, such as public health, or muon science, give each instance a specific flavour for local users. - The analysis needs of any field, can be satisfied, by installing a matching set of tools. --- ## Free analysis services drive software quality improvement and scientific value - Galaxy source code has been running on real hardware since the initial 2005 publication. - Providing free, reliable analysis services and storage for anyone, anywhere. - Increasingly heavy use provides realistic, non-stop software stress testing - Users contribute insight and experience to improve functionality - Independent developers contribute improvements and help fix bugs - Reliable software engineering practices are encouraged by heavy use at scale ??? - An innovative analysis service, running Galaxy code, has been available since two thousand and five. - It quickly attracted users, collaborators, and research grant support. - Running a free, and highly used service, provides thorough testing, for the code. - Tens of thousands of scientists generate bug reports,feedback and ideas for improvement - Participants contribute bug fixes, ideas and code improvements. - This helps us maintain robust code, that provides increasing value to users, over time. --- ## Organic, collaborative growth driven by increasing value to scientists - Reliable, free analysis services rapidly attracted users and contributors. - New interacting grants have joined and extended the project, such as the GTN - Growth is organic and self-regulating. - Open shared values make it easier for large scale collaborations to function efficiently. - Scientists benefit from easy access to reliable complex analyses and training - Hundreds of community participants also regularly contribute their skills and ideas. ???
