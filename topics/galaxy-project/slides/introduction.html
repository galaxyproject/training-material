- The context in which Galaxy originated, in 2006, helps explain a lot. - Sequencing was expensive but provided valuable insights. - Technical skills required to analyse it correctly were in very short supply - Investigators were impatient for results to publish from their expensive data. - If the analysis cannot be scrutinised, the results cannot gain scientific trust. - Galaxy was our response to this growing shortfall in reliable analysis. - Not just a crisis in genome sciences, but in all other growing data rich fields. --- ## Solution: Accessible analysis using shareable and repeatable computation. - Galaxy was created as a solution to these growing problems. - Integrate sharing and computational reproducibility into a generic analysis framework. - XML wrapper turns any Linux command line package into an interoperable tool - Wrapped packages gain framework benefits, such as workflow inter-operability - ToolShed repository automates tool wrapper installation for server administrators. - "Flavour" each framework instance with specific tools to suit the needs of local users. - Adaptable to any kind of open science ??? - Galaxy was created to make open science analyses reproducible and transparent. - Solve those hard challenges with a generic reproducible analysis application. - Build those core scientific requirements into every analysis and output. - Accommodate any open source analysis package, to suit any scientist. - Make it easy to create and add tools to the generic framework. - Each tool combination creates a user-specific analysis platform "flavour". - Local users see only that selection of tools for their specific needs. --- ## Free analysis services drive software quality improvement and scientific value - Since 2006, the Galaxy project has deployed the source code on real hardware. - Providing free, reliable analysis services and storage for anyone, anywhere. - Increasingly heavy use provides realistic, non-stop software stress testing - Users contribute insight and experience to improve functionality - Independent developers contribute improvements and help fix bugs - Reliable software engineering practices are encouraged by heavy use at scale ??? - Galaxy code was available in 2006. - An innovative analysis service running that code was also supported. - It quickly attracted users collaborators and research grant support. - Running the code as a free and highly reliable service is an impressive feat. - Tens of thousands of scientists use it every day now. - Thoroughly stress tested. - Highly responsive to real world user feedback. --- ## Organic, collaborative growth driven by increasing value to scientists - Reliable, free analysis services rapidly attracted users and contributors. - New interacting grants have joined and extended the project, such as the GTN - Growth is organic and self-regulating. - Open shared values make it easier for large scale collaborations to function efficiently. - Scientists benefit from easy access to reliable complex analyses and training - Hundreds of community participants also regularly contribute their skills and ideas. ???
