- The context in which Galaxy originated, in two thousand and five, helps explain a lot. - Sequencing was expensive, but provided valuable scientific insights. - Technical skills needed to analyse raw data correctly, were in very short supply - Investigators were impatient for results to publish from expensive data. - If an analysis cannot be scrutinised, the results cannot gain scientific trust. - Galaxy was our response to this growing shortfall in reliable analysis capacity. - Not only in genome sciences, but a risk to scientific integrity in many rapidly growing, data-rich fields. --- ## Solution: Accessible analysis using shareable and repeatable computation. - Galaxy was created as a solution to these growing problems. - Integrate sharing and computational reproducibility into a generic analysis framework. - XML wrapper turns any Linux command line package into an interoperable tool - Wrapped packages gain framework benefits, such as workflow inter-operability - ToolShed repository automates tool wrapper installation for server administrators. - "Flavour" each framework instance with specific tools to suit the needs of local users. - Adaptable to any kind of open science ??? - Galaxy was created to make open science analyses reproducible and transparent. - Solve those hard challenges, with a generic reproducible analysis application. - Build those core scientific requirements into every analysis and output. - Accommodate any open source analysis package, to suit any scientist. - Make it easy to wrap, and install, new tools into that generic framework. - Each framework instance acquires a flavour, when tools for specific analyses are installed. - So the analysis needs of any field can be satisfied, by installing a matching set of tools. --- ## Free analysis services drive software quality improvement and scientific value - Galaxy source code has been running on real hardware since the initial 2005 publication. - Providing free, reliable analysis services and storage for anyone, anywhere. - Increasingly heavy use provides realistic, non-stop software stress testing - Users contribute insight and experience to improve functionality - Independent developers contribute improvements and help fix bugs - Reliable software engineering practices are encouraged by heavy use at scale ??? - An innovative analysis service running Galaxy code has been supported since 2005. - It quickly attracted users, collaborators, and research grant support. - Running a free, and highly reliable service is a thorough test of the code. - Tens of thousands of scientists use it every day now. - They provide feedback and ideas for improvement - This helps maintain robust code and increasing value to users. --- ## Organic, collaborative growth driven by increasing value to scientists - Reliable, free analysis services rapidly attracted users and contributors. - New interacting grants have joined and extended the project, such as the GTN - Growth is organic and self-regulating. - Open shared values make it easier for large scale collaborations to function efficiently. - Scientists benefit from easy access to reliable complex analyses and training - Hundreds of community participants also regularly contribute their skills and ideas. ???
