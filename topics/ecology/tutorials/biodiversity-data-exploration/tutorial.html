Biodiversity data exploration

Authors: Olivier Norvez Marie Coline Royaux Yvan Le Bras hall-of-fameAdd Contributions!

Overview

Questions:

  • How to explore biodiversity data?

  • How to look at Homoscedasticity, normality or collinearity of presences-absence or abundance data?

  • How to compare beta diversity taking into account space, time and species components?

Introduction

This tutorial will guide you on the exploration of biodiversity data having taxonomic, spatial and temporal informations.

We‚Äôll be using Reef Life Survey (RLS) data extracted from the Australian Ocean Data Network (AODN) portal. We‚Äôll use a subset done directly on the AODN data portal (https://portal.aodn.org.au/) on this dataset ‚ÄúIMOS - National Reef Monitoring Network Sub-Facility - Global reef fish abundance and biomass‚Äù. We decided to use data only on the Mollusca phylum from the east coast of Australia between 2008 and 2021. We‚Äôll explore this dataset in the view of making statistical analyses so we will check the homoscedasticity and normality of the variables, see if some variables are correlated, how the data is distributed through space and time, etc ‚Ä¶ And finally, we‚Äôll explore Beta diversity through the computation of the SCBD and LCBD (Species and Local Contribution to Beta Diversity).

details Definitions of SCBD and LCBD

Species Contribution to Beta Diversity: degree of variation for individual species across the study area.

Local Contribution to Beta Diversity: comparative indicators of the ecological uniqueness of the sites.

Data preparation

First step is to download biodiversity data on your Galaxy history. Here we will use a ‚Äúclassical‚Äù (containing taxonomic, spatial and temporal informations) biodiversity dataset from the well known ‚ÄúReef life survey‚Äù initiative.

Get data

hands_on Hands-on: Data upload

  1. Create a new history for this tutorial and give it a name (example: ‚ÄúRLS for biodiversity data exploration tutorial‚Äù) for you to find it again later if needed.

  2. Import the files from Zenodo

    Tip: Importing via links

    • Copy the link location

    • Open the Galaxy Upload Manager (galaxy-upload on the top-right of the tool panel)

    • Select Paste/Fetch Data

    • Paste the link into the text field

    • Press Start

    • Close the window

    Tip: Importing data from a data library

    As an alternative to uploading the data from a URL or your computer, the files may also have been made available from a shared data library:

    • Go into Shared data (top panel) then Data libraries
    • Navigate to the correct folder as indicated by your instructor
    • Select the desired files
    • Click on the To History button near the top and select as Datasets from the dropdown menu
    • In the pop-up window, select the history you want to import the files to (or create a new one)
    • Click on Import

  3. Rename the datasets ‚Äúreef_life_molluscs‚Äù for example and preview your dataset

    You can see that the dataset hasn‚Äôt been detected to be a CSV dataframe, it is because RLS data directly puts the metadata of the dataframe in the first lines before the dataframe so you‚Äôll have to remove these lines using the Remove beginning Tool: Remove beginning1 with the following parameters: - param-text ‚ÄúRemove first‚Äù: 72 - param-file ‚Äúfrom‚Äù: reef_life_molluscs data file

    Then, verify if your new file hasn‚Äôt got hashtags in the first lines and then ask Galaxy to autodetect datatype (click on the pencil, then ‚ÄúDatatypes‚Äù then click on ‚ÄúAuto-detect‚Äù button). Galaxy will normally detect it as csv.

  4. Convert datatype CSV to tabular

    Tip: Converting the file format

    • Click on the galaxy-pencil pencil icon for the dataset to edit its attributes
    • In the central panel, click on the galaxy-gear Convert tab on the top
    • Select Convert CSV to tabular
    • Click the Convert datatype button

In order to clean unnecessary informations from the table we will now cut a few columns and change the format of time information.

hands_on Hands-on: Clean your data

  1. Use Advanced cut columns from a table (cut) Tool: toolshed.g2.bx.psu.edu/repos/bgruening/text_processing/tp_cut_tool/1.1.0 with following parameters :
    • param-files ‚ÄúFile to cut‚Äù: Convert CSV to tabular data file
    • param-select ‚ÄúOperation‚Äù: Keep
    • param-select ‚ÄúDelimited by‚Äù: Tab
    • param-select ‚ÄúCut by‚Äù: fields
      • param-select ‚ÄúList of Fields‚Äù: Column: 8 Column: 10 Column: 11 Column: 12 Column: 25 Column: 28
  2. Use Column Regex Find And Replace Tool: toolshed.g2.bx.psu.edu/repos/galaxyp/regex_find_replace/regexColumn1/1.0.0 with following parameters:
    • param-files ‚ÄúSelect cells from‚Äù: Advanced Cut data file
    • param-select ‚Äúusing column‚Äù: Column: 4
    • param-repeat Click ‚Äù+ Insert Check‚Äù:
      • param-text ‚ÄúFind Regex‚Äù: ([0-9]{4})-[0-9]{2}-[0-9]{2}
      • param-text ‚ÄúReplacement‚Äù: \1

Data checking

Homoscedasticity and normality analysis

hands_on Hands-on: Here we will check homogeneity of variances (Levene test) for every species and represent it through multiple boxplots and the normal distribution (Kolmogorov-Smirnov test) represented by a distribution histogram and a Q-Q plot.

  1. Homoscedasticity and normality Tool: toolshed.g2.bx.psu.edu/repos/ecology/ecology_homogeneity_normality/ecology_homogeneity_normality/0.0.0 with the following parameters:
    • param-file ‚ÄúInput table‚Äù: Column Regex Find and Replace data file
    • param-select ‚ÄúFirst line is a header line‚Äù: Yes
    • param-select ‚ÄúSelect column containing temporal data (year, date, ‚Ä¶)‚Äù: c4
    • param-select ‚ÄúSelect column containing species‚Äù: c5
    • param-select ‚ÄúSelect column containing numerical values (like abundances)‚Äù: c6

    You have to get three outputs: the Levene Test for homoscedasticity dataset, the Kolmogrov-Smirnov test for normality and 9 PNG files in a data collection representing the homogeneity of variances for each species at each time point of the study. If the levene test is significant (P-value in column Pr < 0.5 and at least one * at the end of the 4th line), variances aren‚Äôt homogeneous, the hypothesis of homoscedasticity is rejected. If the K-S test is significant (p-value < 0.5), your numerical variable isn‚Äôt normally distributed, the hypothesis of normality is rejected. The two tests have to be significant so variances aren‚Äôt homogenous and data isn‚Äôt normally distributed.

Autocorrelation in your data

hands_on Hands-on: Autocorrelation

  1. Variables exploration Tool: toolshed.g2.bx.psu.edu/repos/ecology/ecology_link_between_var/ecology_link_between_var/0.0.0 with the following parameters:
    • param-file ‚ÄúInput table‚Äù: Column Regex Find and Replace data file
    • param-select ‚ÄúFirst line is a header line‚Äù: Yes
    • param-select ‚ÄúVariables links exploration‚Äù: Autocorrelation of one selected numerical variable
      • param-select ‚ÄúSelect columns containing numerical values‚Äù: c6

    You have to get two outputs, one text file containing the Autocorrelation function values and one PNG file in the data collection showing the autocorrelation for a variable. If the bars of the histogram are strictly confined between the dashed lines (representing 95% confidence interval without white noise), there is auto-correlation.

    Here, we don‚Äôt see there is autocorrelation.

    • param-file ‚ÄúInput table‚Äù: formatted biodiversity data file
    • param-select ‚ÄúFirst line is a header line‚Äù: Yes
    • param-select ‚ÄúVariables links exploration‚Äù: Collinearity between selected numerical variables for each species
      • param-select ‚ÄúSelect column containing species‚Äù: c5
      • param-select ‚ÄúSelect columns containing numerical values‚Äù: c['4', '6']

    You have to get two outputs, one describing species we couldn‚Äôt evaluate and one PNG file with one plot containing multiple correlation plots and the correlation values between each variables.

Data exploration

Visualize abundance repartition through space

hands_on Hands-on: Abundance map in the environment

  1. Presence-absence and abundance Tool: toolshed.g2.bx.psu.edu/repos/ecology/ecology_presence_abs_abund/ecology_presence_abs_abund/0.0.0 with the following parameters:
    • param-file ‚ÄúInput table‚Äù: formatted biodiversity data file
    • param-select ‚ÄúFirst line is a header line‚Äù: Yes
    • param-select ‚ÄúVariables presence, absence and abundance‚Äù: Abundance map in the environment
      • param-select ‚ÄúSelect column containing latitudes ‚Äú: c2
      • param-select ‚ÄúSelect column containing longitudes‚Äù: c3
      • param-text ‚ÄúWhat do you study in this analysis ?‚Äù: Molluscs of Australian east coast
      • param-select ‚ÄúSelect column containing taxon ‚Äú: c5
    • param-select ‚ÄúSelect column containing abundances ‚Äú: c6

    You have to get two outputs, one with the map of the abundance through space with the coordinates and one text file to inform you about the geographical extent of your map.

    • param-file ‚ÄúInput table‚Äù: formatted biodiversity data file

    • param-select ‚ÄúSelect column containing abundances ‚Äú: c6

    You have to get two outputs, one with 120 PNG files (one for each site) representing the number of locations where each taxons are present and one text file to inform you about the used locations.

    • param-file ‚ÄúInput table‚Äù: formatted biodiversity data file

Visualize the dispersion of a numeric variable and the correlation between species absence

hands_on Hands-on: Boxplot of dispersion and correlation of absence

  1. Statistics on presence-absence Tool: toolshed.g2.bx.psu.edu/repos/ecology/ecology_stat_presence_abs/ecology_stat_presence_abs/0.0.0 with the following parameters:
    • param-file ‚ÄúInput table‚Äù: formatted biodiversity data file
    • param-select ‚ÄúFirst line is a header line‚Äù: Yes
    • param-select ‚ÄúSelect a column containing numerical values (such as the abundance) ‚Äú: c6
    • param-select ‚ÄúSelect the column of the x-axis : most commonly species‚Äù: c5
    • param-select ‚ÄúSelect column containing locations ‚Äú: c1
    • param-select ‚ÄúSelect column containing temporal data (year, date, ‚Ä¶) ‚Äú: c4

    You have to get two outputs, one PNG file with the boxplot and dispersion plot of the abundance and one plot representing wether the absence of several species is correlated. In the second plot if you see there is a cross on the round representation the two related species haven‚Äôt got their absences correlated, the other are correlated and seem to be co-absent.

Beta diversity

Local and Species Contribution to Beta Diversity (SCBD and LCBD)

hands_on Hands-on: Task description

  1. Local Contributions to Beta Diversity (LCBD) Tool: toolshed.g2.bx.psu.edu/repos/ecology/ecology_beta_diversity/ecology_beta_diversity/0.0.0 with the following parameters:
    • param-file ‚ÄúInput table‚Äù: formatted biodiversity data file
    • param-select ‚ÄúFirst line is a header line‚Äù: Yes
    • param-select ‚ÄúSelect column with abundances‚Äù: c6
    • param-select ‚ÄúSelect column with locations‚Äù: c1
    • param-select ‚ÄúSelect column containing taxon‚Äù: c5
    • param-select ‚ÄúSelect column containing dates‚Äù: c4
    • param-select ‚ÄúOther LCBD : spatialized representation or xy plot.‚Äù: Spatialized representation
      • param-select ‚ÄúSelect column containing latitudes in decimal degrees‚Äù: c2
      • param-select ‚ÄúSelect column containing longitudes in decimal degrees‚Äù: c3

    You have to get three outputs. Two text file containing a table with information on the beta diversity and one text file with the list of species that has a SCBD larger than the mean SCBD. One data collection with PNG files showing multiple plots according to one type of variable in order to vizualize the beta diversity.
