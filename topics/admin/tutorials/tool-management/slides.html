- Yes: Query install DB for `repo_owner`, `repo_name`, `changeset_id`, then source: - `/bar/4.2////env.sh` - No: continue --- class: left ## `` Does `/bar/4.2/env.sh` exist?: - Yes: source it - No: Does `/bar/4.2/bin/` exist?: - Yes: prepend it to `$PATH` - No: continue Good for dependencies manually installed by an administrator --- class: left ## `` Does `bar==4.2` exist in Conda?: - Yes: use `conda create` to create a Conda env in `/envs/` (if it doesn't exist already), then activate it - No: continue [Conda for tool dependencies FAQ](https://docs.galaxyproject.org/en/master/admin/conda_faq.html) --- class: left ## `` Does `/bar/default/` exist and is it a symlink?: - Yes: Does `/bar/default/env.sh` exist?: - Yes: source it - No: Does `/bar/default/bin/` exist?: - Yes: prepend it to `$PATH` - No: continue - No: continue --- class: left ## `` Does `bar` (any version) exist in Conda?: - Yes: use `conda create` to create env in job working directory - No: continue --- class: left ## `` Is `bar/4.2` a registered module?: - Yes: Load module `bar/4.2` - No: continue See: [Environment Modules](http://modules.sourceforge.net/) --- class: left ## `` Is `bar/default` a registered module?: - Yes: Load module `bar/default` - No: continue --- ## If dependency is not resolved Submit the job anyway, maybe it's available on `$PATH` --- ## Using containers for Tool Dependencies - Galaxy can also use **Docker** or **Singularity** containers to resolve dependencies - You can read more about this in the [Tool Dependencies and Containers slides](../../../dev/tutorials/containers/slides.html)
